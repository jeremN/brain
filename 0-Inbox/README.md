# 0-Inbox

Capture queue. Anything goes here when you don't have time to file it properly.

## Rule

**Nothing stays in `0-Inbox/` longer than 2 weeks.** If it's still here after 2 weeks, either:
- File it properly (`3-Resources/<topic>/`, `1-Projects/<project>/`, `2-Areas/<area>/`)
- Delete it (it wasn't important)

## Contents

- `urls-to-process.md` — URL capture queue. The `process-veille` skill drains this into fiches.
- `clippings/` — rough notes, screenshots, ideas not yet structured. (create this folder when you need it)

## Triage workflow

When you sit down for a weekly review:

1. Drain `urls-to-process.md` (run the skill).
2. Move/rewrite anything in `clippings/` into the right PARA folder.
3. Inbox should be empty at the end of the review.
