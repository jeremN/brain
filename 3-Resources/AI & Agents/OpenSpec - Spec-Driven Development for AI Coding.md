# OpenSpec - Spec-Driven Development for AI Coding

**Source:** [github.com/Fission-AI/OpenSpec](https://github.com/Fission-AI/OpenSpec)
**Category:** AI & Agents

## Summary

OpenSpec est un framework de développement piloté par les spécifications (SDD) pour les assistants de code IA. Il ajoute une couche de spécification légère entre le développeur et l'agent, structurant le travail en cycle propose → spec → design → implement → archive. Compatible avec 20+ assistants IA via des slash commands (`/opsx:propose`, `/opsx:apply`), il permet d'aligner les requirements avant d'écrire du code, sans imposer de phases rigides.

## Why It's Interesting

Complémentaire aux outils de context engineering comme GSD : là où GSD optimise l'exécution technique du LLM, OpenSpec se concentre sur la qualité des spécifications en amont. L'approche "fluid not rigid" permet d'itérer sur n'importe quel artifact sans phase gates.

## Key Takeaways

- Workflow artifact-guided : proposals, specs, design docs et tasks organisés par dossier de changement
- Slash commands intégrés aux assistants IA (Claude, ChatGPT, etc.)
- Itération flexible — pas de waterfall, modification de n'importe quel artifact à tout moment
- Tool-agnostic : npm, pnpm, yarn, bun supportés
- 33k+ stars, 50+ contributeurs, MIT license — projet activement maintenu
