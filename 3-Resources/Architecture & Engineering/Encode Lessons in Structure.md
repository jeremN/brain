# Encode Lessons in Structure

**Source:** [https://github.com/poteto/noodle/blob/main/brain/principles/encode-lessons-in-structure.md](https://github.com/poteto/noodle/blob/main/brain/principles/encode-lessons-in-structure.md)
**Category:** Architecture & Engineering

## Summary
An architectural principle about embedding lessons learned and domain knowledge directly into code structure and organization. Makes implicit knowledge explicit through system design.
## Why It's Interesting
Shifts focus from code as instructions to code as knowledge documentation.
## Key Takeaways
- Code structure should reflect domain knowledge
- Prevents re-learning lessons from experience
- Guides future developers and maintainers
- Improves consistency and predictability
