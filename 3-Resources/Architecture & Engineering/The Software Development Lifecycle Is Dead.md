# The Software Development Lifecycle Is Dead

**Source:** [https://boristane.com/blog/the-software-development-lifecycle-is-dead/](https://boristane.com/blog/the-software-development-lifecycle-is-dead/)
**Category:** Architecture & Engineering

## Summary

Boris Tane argues AI agents haven't made the SDLC faster — they've killed it. The seven traditional stages collapse into a tight loop: Intent -> Agent -> Build+Test+Deploy -> Observe -> Repeat. AI-native engineers who started after Cursor launched don't know what DevOps or SRE is. The only surviving stage is monitoring, which becomes the primary safety mechanism and feedback loop for the system.

## Why It's Interesting

AI-native engineers who started post-Cursor have never done sprint planning, never estimated story points — and they're unencumbered by it

## Key Takeaways

- The SDLC stages didn't get faster, they merged — the agent doesn't know what step it's on because there are no steps
- Requirements are now a byproduct of iteration, not a phase — generate ten versions and pick the best one
- System design is discovered through agent collaboration, not dictated ahead of time
- Code review as PR workflow is becoming a bottleneck: agent generates 500 PRs/day, team reviews maybe 10
- Monitoring is the last stage standing — observability feeds back into the agent loop, not a human's pager
- The new skill is context engineering; the new safety net is observability
- Jira becomes a context store (and a terrible one) when agents, not humans, are the consumers
