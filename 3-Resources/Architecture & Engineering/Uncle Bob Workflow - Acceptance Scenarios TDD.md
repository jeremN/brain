# Workflow d'Uncle Bob : Scénarios d'Acceptation et TDD

**Source:** [@unclebobmartin](https://x.com/unclebobmartin/status/2032145477786099912)
**Category:** Architecture & Engineering

## Résumé
Uncle Bob Martin propose un workflow efficace pour le développement : pour chaque nouveau comportement ou changement, écrire d'abord les scénarios d'acceptation. Avant de modifier le comportement existant, il faut d'abord demander (interroger les besoins). Cette approche s'inscrit dans une philosophie TDD rigoureuse.

## Pourquoi c'est intéressant
Ce workflow améliore la qualité du code et réduit les bugs en établissant une spécification claire avant l'implémentation. La clarification préalable des changements de comportement existant prévient les régressions accidentelles.

## Points clés
- **Scénarios d'acceptation d'abord** - Définir le comportement attendu avant de coder
- **Demander avant de changer** - Clarifier les impacts sur le comportement existant
- **TDD rigoureux** - Approche disciplinée et éprouvée
