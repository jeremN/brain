**Source:** [@adxtyahq on X](https://x.com/adxtyahq/status/2021974597256843341)
**Category:** Architecture & Engineering

## Résumé
Un regard sur les règles drastiques que la NASA impose pour le code C critique mission - y compris l'interdiction de la récursion et autres contraintes sévères. C'est une fenêtre sur comment l'industrie critique conçoit la fiabilité absolue.

## Pourquoi c'est intéressant
Les standards NASA représentent l'extrême de la rigueur ingénierie. Même si impratiques pour la plupart des projets, les principes sous-jacents offrent des insights sur l'importance de la contrainte, clarté et prévisibilité.

## Points clés
- **No recursion** - Contrainte radicale sur la complexité
- **Mission-critical** - Zéro tolérance pour l'erreur
- **NASA standards** - Rigueur extrême de l'agence spatiale
- **Safety engineering** - Priorité absolue: fiabilité
