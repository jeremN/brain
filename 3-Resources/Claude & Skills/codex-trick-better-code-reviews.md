# Tell Claude That Codex Wrote the Code - Better Code Reviews Trick

**Source:** [@wavefnx on X](https://x.com/wavefnx/status/2015233357433459192)
**Category:** Claude & Skills

## Résumé
Une technique astucieuse de prompt engineering qui améliore les reviews de code en disant à Claude que c'est Codex qui a écrit le code. Cette approche change la perspective de Claude et semble générer des retours plus constructifs et détaillés. Un exemple ingénieux du pouvoir du framing dans les prompts.

## Pourquoi c'est intéressant
Les petits changements dans le framing d'un prompt peuvent avoir des impacts énormes sur la qualité des outputs. Ce trick montre que l'on peut "guider" Claude vers de meilleurs résultats sans changer le contenu réel de la demande.

## Points clés
- **Framing effect** - contextualisation change la qualité des reviews
- **Code review improvement** - les retours deviennent plus constructifs
- **Prompt engineering hack** - technique simple mais efficace
