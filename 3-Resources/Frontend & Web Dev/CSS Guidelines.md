# CSS Guidelines

**Source:** [https://cssguidelin.es/](https://cssguidelin.es/)
**Category:** Frontend & Web Dev
**Original Section:** CSS / Guidelines

## Summary

Harry Roberts' comprehensive, industry-standard guide for writing maintainable, scalable CSS. Covers BEM naming, file organization, specificity management, and ITCSS architecture. The reference for CSS architecture in enterprise projects.

## Why It's Interesting

The definitive CSS architecture reference that influenced many design systems and CSS methodologies

## Key Takeaways

- Keep specificity low and flat — avoid nesting selectors deeply or using IDs for styling
- Use BEM naming convention for predictable, self-documenting class names
- Organize CSS into settings, tools, generic, elements, objects, components, utilities (ITCSS)
- Never use !important reactively — only proactively in utility classes
- Write CSS that other developers can understand and extend without fear
