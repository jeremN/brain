# Chrome passe à un cycle de release de deux semaines

**Source:** [https://developer.chrome.com/blog/chrome-two-week-release](https://developer.chrome.com/blog/chrome-two-week-release?hl=fr)
**Category:** Frontend & Web Dev

## Summary
À partir de septembre 2026 (Chrome 153), Google passe d'un cycle de 4 semaines à 2 semaines pour les releases stable et beta. L'objectif : donner aux développeurs et utilisateurs un accès immédiat aux dernières améliorations.

## Why It's Interesting
Changement majeur dans le rythme du web — des releases plus fréquentes mais plus petites, facilitant le debugging post-release et réduisant les disruptions.

## Key Takeaways
- **Chrome 153 (8 sept 2026)** : première release du nouveau cycle
- **Toutes plateformes** : Desktop, Android, iOS
- **Extended Stable** : reste à 8 semaines
- **Dev et Canary** : pas de changement
- **Releases plus petites** = scope réduit, debugging simplifié
