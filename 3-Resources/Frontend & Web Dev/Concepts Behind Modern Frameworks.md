# Concepts Behind Modern Frameworks

**Source:** [https://dev.to/lexlohr/concepts-behind-modern-frameworks-4m1g](https://dev.to/lexlohr/concepts-behind-modern-frameworks-4m1g)
**Category:** Frontend & Web Dev
**Original Section:** To Classify

## Summary

Deep dive into core concepts powering modern JS frameworks: reactivity (signals, proxies, dirty checking), rendering strategies (virtual DOM, fine-grained updates, compiled output), and component models. Explains why frameworks made different trade-offs.

## Why It's Interesting

Framework-agnostic education that helps you understand the 'why' behind design choices

## Key Takeaways

- Reactivity: signals (Solid, Angular), proxies (Vue), compiler-driven (Svelte), virtual DOM diffing (React)
- Fine-grained reactivity updates only what changed without diffing — signals increasingly popular
- Compiled frameworks shift work from runtime to build time for better performance
- Understanding these concepts helps evaluate new frameworks without getting caught in hype
- The trend moves toward signals and compile-time optimizations over virtual DOM
