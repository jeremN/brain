# Tao of React

**Source:** [https://alexkondov.com/tao-of-react/](https://alexkondov.com/tao-of-react/)
**Category:** Frontend & Web Dev
**Original Section:** React

## Summary

Alex Kondov's influential guide establishing principles for writing clean, maintainable React code. Covers component design, state management, side effects, and project structure. Advocates for co-location of related code, composing components from small focused pieces, keeping components pure, and avoiding premature abstraction.

## Why It's Interesting

Canonical React philosophy guide that influenced how teams structure large React applications

## Key Takeaways

- Co-locate related code: keep styles, tests, and components that work together in the same directory
- Components should do one thing well — extract when a component does more than one thing
- Don't hardcode markup: use data-driven rendering and composition over static templates
- Keep state as close as possible to where it's used — lift only when necessary
- Favor composition over inheritance and render props over complex prop drilling
