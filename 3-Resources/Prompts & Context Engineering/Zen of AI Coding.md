# Zen of AI Coding

**Source:** [https://nonstructured.com/zen-of-ai-coding/](https://nonstructured.com/zen-of-ai-coding/)
**Category:** Prompts & Context Engineering

## Summary

Philosophical manifesto on AI-assisted software development, structured as principles like Python's Zen. Key thesis: software development as manual code production is dying. The new discipline centers on framing problems, shaping context, defining constraints, and judging outcomes. Covers economics of cheap code, refactoring ease, feedback loops, stack agnosticism, and the shift in buy-vs-build calculus.

## Why It's Interesting

Buy vs Build calculus has flipped: when code approaches zero marginal cost, many services that once required vendors are now within reach

## Key Takeaways

- Code is cheap: the bottleneck shifts to product decisions, security review, user testing, and release processes
- Refactoring is easy: architectural decisions are now provisional — a flawed implementation provides better context than a pristine spec
- All bugs are shallow: multiple AI reviewers find different issues, but the limit is tightness of feedback loops
- Any stack is your stack: attachment to a specific stack becomes less relevant when you're not writing the code
- The context bottleneck is in your head: human coordination is the limiting factor, not tokens
- Build for agents: AX (Agent Experience) is the new UX — expose structured context and machine-readable surfaces
- Software is a liability, a product is an asset — resist adding features just because building is easy
