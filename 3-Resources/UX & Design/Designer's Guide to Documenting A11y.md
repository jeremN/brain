# Designer's Guide to Documenting A11y

**Source:** [https://stephaniewalter.design/blog/a-designers-guide-to-documenting-accessibility-user-interactions/](https://stephaniewalter.design/blog/a-designers-guide-to-documenting-accessibility-user-interactions/)
**Category:** UX & Design
**Original Section:** To Classify

## Summary

Stephanie Walter's guide on documenting accessibility requirements in design deliverables. Covers keyboard navigation, focus management, screen reader behavior, touch targets, motion preferences, and error states with annotation templates.

## Why It's Interesting

Essential reference for design-to-dev handoff that ensures accessibility isn't lost in translation

## Key Takeaways

- Document keyboard navigation order and focus management for every interactive element
- Annotate screen reader announcements: what gets read, in what order, when
- Specify touch target sizes (minimum 44x44px) and spacing between interactive elements
- Include motion/animation notes for prefers-reduced-motion
- Document error, loading, and empty states — not just the happy path
