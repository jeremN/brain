# Details That Make Interfaces Feel Better

**Source:** [https://jakub.kr/writing/details-that-make-interfaces-feel-better](https://jakub.kr/writing/details-that-make-interfaces-feel-better)
**Category:** UX & Design

## Summary

Jakub Linowski's collection of subtle UI/UX polish techniques that make interfaces feel more refined and professional. Covers practical CSS and design patterns: text-wrap balance for headings, concentric border radius (inner radius = outer radius minus padding), interruptible animations that don't feel laggy, optical alignment adjustments, and when to use shadows vs borders for depth. Each technique is illustrated with before/after comparisons.

## Why It's Interesting

Actionable CSS-level techniques that immediately make any interface feel more polished — the kind of details that separate amateur from professional UI work

## Key Takeaways

- text-wrap: balance — CSS property that evens out line lengths in headings, preventing ugly orphan words
- Concentric border radius: inner radius should equal outer radius minus the gap/padding for visually correct nested rounded corners
- Interruptible animations: allow users to reverse/cancel animations mid-flight instead of waiting for completion — reduces perceived latency
- Optical alignment: icons and text often need manual nudges beyond mathematical centering to look visually centered
- Shadows vs borders: shadows provide softer depth cues for floating elements, borders work better for inline separation
