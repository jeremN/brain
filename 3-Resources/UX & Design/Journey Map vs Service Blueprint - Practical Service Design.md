# Journey Map vs Service Blueprint - Practical Service Design

**Source:** [https://medium.com/practical-service-design/the-difference-between-a-journey-map-and-a-service-blueprint-31a6e24c4a6c](https://medium.com/practical-service-design/the-difference-between-a-journey-map-and-a-service-blueprint-31a6e24c4a6c)
**Category:** UX & Design

## Summary

Article de Practical Service Design (Megan Erin Miller & Erik Flowers) qui clarifie la distinction fondamentale entre journey maps et service blueprints. Les deux outils se situent aux extrémités opposées d'un continuum subjectif/objectif.

### La métaphore clé

La journey map est un **récit** ("tale") — elle raconte l'histoire émotionnelle et rationnelle du client. Le service blueprint est un **script** — il décrit la mécanique complète de livraison du service, de la surface jusqu'au cœur.

### Journey Map

- Focus : expérience client, front-stage uniquement
- Perspective : subjective, émotionnelle
- Contenu : phases, actions, pensées, émotions, pain points
- Question : "Que vit le client ?"

### Service Blueprint

- Focus : livraison du service de bout en bout, surface-to-core
- Perspective : objective, opérationnelle
- Contenu : actions client + frontstage + backstage + processus support
- Question : "Comment le service est-il réellement délivré ?"

### Relation entre les deux

La journey map constitue la couche client du service blueprint. Le blueprint l'enrichit en révélant tout ce qui se passe en coulisses pour supporter l'expérience visible. Souvent, la réalité du service diffère de ce que l'organisation pense avoir conçu — le blueprint révèle ces écarts.

## Why It's Interesting

Clarifie une confusion fréquente dans les équipes produit/design. L'approche "Practical" de Miller & Flowers rend le service design actionnable plutôt qu'académique.

## Key Takeaways

- Journey map = récit subjectif du client ; blueprint = script objectif du service
- La journey map est un sous-ensemble du service blueprint (la couche client)
- Le blueprint révèle les écarts entre service conçu et service réel
- Les deux outils sont complémentaires : commencer par la journey map, puis approfondir avec le blueprint
- L'approche Practical Service Design vise des livrables actionnables
